Aufgabe

Übungen - Vorbereitungen

  1. Teilt euch in Teams ein (2 pro Team)

    1. Registrieren bei https://github.com

    2. UserId od. Name PLUS Teammitglied mir mitteilen

  2. Ich weise euch auf Github einer Organisation zu

  3. jeder clont (nicht forken) das Repository progit2 aus seiner Organisation

    • nehmt den Link https://github.com/…​; , nicht git@github.com:…​

    • git clone in einen Ordner ausführen, der bisher nicht mit Git genutzt wurde

Übungen 1

  1. Mitglied 1: ändere mehrere Dateien auf master und publiziere diese Änderungen, sodass sie auf github sichtbar sind

  2. Mitglied 2: hole diese Änderungen auf in deine Workcopy

  3. Mitglied 1 & 2: ändert gleichzeitig mehrere Dateien auf master und publiziere diese Änderungen, sodass sie auf github sichtbar sind

  4. Mitglied 1 & 2: ändert gleichzeitig die gleiche Dateien, die gleiche Zeile auf master und publiziere diese Änderungen, sodass sie auf github Sichtbar sind

Übungen 2

  1. Mitglied 1 & 2: jeder erstellt (auf github) einen neuen Branch von master (siehe help.github.com)

  2. Mitglied 1 & 2: jeder aktiviert lokal seinen erstellten Branch (vorher git fetch nötig)

  3. Mitglied 1 & 2: jeder erstellt mehrere Commits auf seinem Branch und publiziere diese Änderungen, sodass sie auf github sichtbar sind

  4. Mitglied 1 & 2: gemeinsam - integriert die Änderungen eines Branches auf master

    • provoziert einen 3WM per git merge --no-ff

  5. Mitglied 1 & 2: gemeinsam - macht auf dem zweiten Branch einen Rebase auf die neue Spitze von master damit ein späterer Merge diese Branches auf master mit einem Fast-Forward-Merge erfolgen kann

    • führt nach dem Rebase einen Merge auf master per git merge --ff-only aus

Übungen 3

  1. Mitglied 1 & 2: gemeinsam

    1. erstellt einen neuen Branch

    2. aktiviert lokal diesen erstellten Branch (vorher git fetch nötig)

    3. erstellt mehrere Commits auf diesem Branch und publiziere diese Änderungen, sodass sie auf github sichtbar sind

  2. Mitglied 1: erstelle von diesem Branch einen Pull-Request

    • siehe https://help.github.com/articles/creating-a-pull-request/

  3. Mitglied 2: Reviewe diesen Pull-Request, kommentiere ggf. einzelne Änderungen, nachdem alles geklärt wurde, nimm diesen Pull-Request an
